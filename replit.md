# Workspace

## Overview

pnpm workspace monorepo using TypeScript. Each package manages its own dependencies.
Also contains Python scripts for Snowflake connectivity, NLP-based data querying, and ELT pipelines.

## Stack

- **Monorepo tool**: pnpm workspaces
- **Node.js version**: 24
- **Package manager**: pnpm
- **TypeScript version**: 5.9
- **API framework**: Express 5
- **Database**: PostgreSQL + Drizzle ORM
- **Validation**: Zod (`zod/v4`), `drizzle-zod`
- **API codegen**: Orval (from OpenAPI spec)
- **Build**: esbuild (CJS bundle)

## Python / Snowflake Stack

- **Python version**: 3.11
- **Snowflake auth**: Programmatic access token (`authenticator=programmatic_access_token`)
- **NLP primary**: `scripts/cortex_agent_workflow.py` — Snowflake Cortex Code-First Agentic Workflow
- **NLP legacy**: `scripts/nlp_query.py` (direct SQL) + `scripts/nlp_query_mcp.py` (MCP agentic)
- **AI (legacy)**: OpenAI gpt-5.1 via Replit AI Integrations proxy
- **Cortex model**: `mistral-large2` via `SNOWFLAKE.CORTEX.COMPLETE()`
- **Eval**: TruLens with 24 test cases across 5 categories

## Cortex Agent Architecture (v2 — primary)

```
User question (NL)
  ↓
CortexBillingAgent  [scripts/cortex_agent_workflow.py]
  │
  ├── Tool 1: Cortex Analyst REST API
  │     POST /api/v2/cortex/analyst/message
  │     Inline semantic model YAML (no UI, no staged file)
  │     Covers: billing_summary + 4 source tables + 6 aggregate tables
  │     → Interprets question → generates Snowflake SQL
  │
  ├── Tool 2: Snowflake SQL Executor
  │     snowflake.connector (PAT auth)
  │     → Runs SQL against cpt_demo.medical → rows + columns
  │
  └── Tool 3: Cortex Complete
        SNOWFLAKE.CORTEX.COMPLETE('mistral-large2', ...)
        → Plain-English explanation citing actual data
```

All three tools are declared in Python code — no Snowflake UI configuration.
Semantic model YAML is a constant in `cortex_agent_workflow.py`.

## ELT Pipeline Architecture

```
expand_dataset.py          — loads 15 patients, 20 CPT codes, 30 visits, 73 procedures
       ↓
cortex_elt_pipeline.py     — CTAS for 6 aggregate tables
       ↓
cortex_elt_pipeline.py     — SNOWFLAKE.CORTEX.COMPLETE() generates 6 narrative insights
       ↓
cortex_insights table      — stored in Snowflake
       ↓
GET /api/insights/data     — serves all agg tables + insights as JSON to UI
POST /api/insights/refresh — re-runs pipeline (with optional expand flag)
       ↓
/insights UI page          — Dashboard / Data Tables / Cortex AI Insights tabs
```

## Key Commands

### Node.js
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- `pnpm --filter @workspace/api-server run dev` — run API server locally

### Python (run from `scripts/`)
- `python expand_dataset.py` — load expanded 15-patient dataset into Snowflake
- `python cortex_elt_pipeline.py` — run full ELT (6 agg tables + 6 Cortex AI insights)
- `python cortex_elt_pipeline.py --json` — dump all aggregate data as JSON (used by API)
- `python cortex_agent_workflow.py` — interactive Cortex Agent REPL (recommended)
- `python cortex_agent_workflow.py --question "..."` — single-shot CLI
- `python cortex_agent_workflow.py --json "..."` — JSON output mode
- `python trulens_eval.py` — full TruLens eval (all 24 cases)
- `python trulens_eval.py --category "Cortex Agent"` — Cortex-only eval
- `python nlp_query.py` — legacy direct SQL pipeline
- `python nlp_query_mcp.py` — legacy MCP pipeline
